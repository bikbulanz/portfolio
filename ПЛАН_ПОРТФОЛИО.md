# План: Сайт-портфолио Руслана Бикбулатова

## Что такое vibe coding (вайбкодинг)

**Vibe coding** — это когда ты не пишешь весь код вручную, а **общаешься с ИИ** (например, Cursor): описываешь, что хочешь, по кусочкам, смотр直播 результат и говоришь «измени это», «добавь то», «сделай проще». Код растёт итеративно, по ощущениям и вкусу.

Для новичка в разработке это идеально: не нужно сразу знать HTML/CSS/JS — достаточно понимать **общую структуру** и уметь формулировать желания на русском или английском.

## Этапы работы

### Этап 1. Подготовка и структура (1–2 дня)

**Цель:** понять, что у нас есть и как это превратить в сайт.

- [ ] **Проверить экспорт Notion** — все ли изображения и ссылки сохранились.
- [ ] **Собрать контент** в одном месте:
  - Фото, логотипы проектов
  - Тексты (о себе, проекты, «что делал»)
  - Ссылки: Telegram, Email, страницы проектов
- [ ] **Определить структуру страниц:**
  - Главная: о себе, контакты, карточки проектов
  - Отдельная страница на каждый проект (по желанию — или всё на одной с раскрытием)
- [ ] **Выбрать технологии:**
  - Простой вариант: один HTML-файл + CSS (можно обойтись без JavaScript)
  - Чуть сложнее: несколько HTML-страниц (главная + страницы проектов)

---

### Этап 2. Каркас сайта (2–3 дня)

**Цель:** «скелет» страницы, чтобы видеть результат в браузере.

- [ ] Создать `index.html` — разметка главной страницы
- [ ] Разметить основные блоки:
  - Шапка (имя, роль, контакты)
  - Секция «О себе»
  - Секция «Проекты» (карточки)
- [ ] Подключить стили в отдельном `style.css` или внутри `<style>` — на твой выбор
- [ ] Добавить контент из Notion (тексты, ссылки)

**Формат задач для ИИ:**  
*«Сделай шапку с именем, ролью и ссылками на Telegram и Email»*  
*«Добавь секцию с карточками проектов — 6 штук, в каждом: картинка, название, краткое описание, период»*

---

### Этап 3. Внешний вид и стиль (2–4 дня)

**Цель:** сделать сайт приятным глазу, в духе продуктового дизайна.

- [ ] Выбрать цветовую схему (например: тёмный фон / светлый текст или наоборот)
- [ ] Подобрать шрифты (Google Fonts — бесплатно)
- [ ] Оформить карточки проектов (скругления, тени, hover-эффекты)
- [ ] Адаптация под мобильные (чтобы было читаемо на телефоне)

**Формат задач для ИИ:**  
*«Сделай карточки проектов с hover-эффектом при наведении»*  
*«Подбери шрифт и цвета в стиле минимализма»*  
*«Сделай, чтобы на мобильном карточки шли в одну колонку»*

---

### Этап 4. Страницы проектов (1–2 дня)

**Цель:** дать возможность углубиться в детали каждого проекта.

- [ ] Решить: отдельные страницы или раскрывающиеся блоки на главной
- [ ] Создать страницы (например: `samoylet.html`, `swiftbook.html` и т.д.)
- [ ] Перенести текст «Что делал в…» и изображения из экспорта Notion
- [ ] Сделать навигацию назад на главную

**Формат задач для ИИ:**  
*«Создай страницу проекта Самолет Плюс с текстом и картинкой»*  
*«Добавь кнопку „Назад“ на страницы проектов»*

---

### Этап 5. Финализация и деплой (1–2 дня)

**Цель:** выложить сайт в интернет.

- [ ] Проверить все ссылки и изображения
- [ ] Сжать изображения (если они тяжелые)
- [ ] Выбрать хостинг:
  - **GitHub Pages** — бесплатно, простой вариант
  - **Vercel / Netlify** — тоже бесплатно, легко заливать
- [ ] Загрузить проект и получить ссылку

**Формат задач для ИИ:**  
*«Как выложить статический сайт на GitHub Pages?»*  
*«Напиши короткую инструкцию по деплою на Vercel»*

---

## Твоя роль в vibe coding

1. **Чётко формулировать** — что хочешь видеть (блок, цвет, расположение).
2. **Смотреть результат** — открывать файл в браузере, проверять.
3. **Давать обратную связь** — «сделай выше», «убери это», «добавь тень».
4. **Разбивать на шаги** — не «сделай весь сайт», а «сделай шапку», потом «добавь карточки».
