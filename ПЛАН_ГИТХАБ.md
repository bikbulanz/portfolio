# План: выложить портфолио на GitHub

## Что получим в итоге

- Репозиторий на GitHub с твоим портфолио
- Возможность хранить и обновлять код в облаке
- Бесплатный хостинг сайта через GitHub Pages (сайт будет доступен по адресу `https://твой-username.github.io/имя-репозитория/`)

---

## Этап 1. Подготовка к Git (5 мин)

### 1.1 Создать файл `.gitignore`

Чтобы в репозиторий не попадали лишние файлы: системные, временные, конфиги редактора.

### 1.2 Инициализировать Git в проекте

Команды:
```bash
cd "путь/к/Руслан Бикбулатов"
git init
```

### 1.3 Первый коммит

```bash
git add .
git commit -m "Первая версия портфолио"
```

---

## Этап 2. Создать репозиторий на GitHub (3 мин)

1. Зайди на [github.com](https://github.com)
2. Войди в аккаунт (или создай новый)
3. Нажми зелёную кнопку **New** (или **Create repository**)
4. Заполни:
   - **Repository name:** `portfolio` (или `ruslan-bikbulatov`, `my-portfolio` — на твой выбор)
   - **Description:** `Портфолио продуктового дизайнера`
   - **Visibility:** Public
   - **НЕ** ставь галочки «Add a README» и «Add .gitignore» — они уже будут в проекте
5. Нажми **Create repository**

---

## Этап 3. Связать локальный проект с GitHub (2 мин)

GitHub покажет команды. Нужно выполнить:

```bash
git remote add origin https://github.com/ТВОЙ_USERNAME/ИМЯ_РЕПОЗИТОРИЯ.git
git branch -M main
git push -u origin main
```

**Важно:** замени `ТВОЙ_USERNAME` и `ИМЯ_РЕПОЗИТОРИЯ` на свои значения.

При первом `git push` тебя попросят войти в GitHub (через браузер или токен).

---

## Этап 4. Включить GitHub Pages (2 мин)

1. В репозитории на GitHub открой **Settings**
2. Слева найди **Pages**
3. В разделе **Source** выбери:
   - Branch: `main`
   - Folder: `/ (root)`
4. Сохрани (**Save**)

Через 1–2 минуты сайт будет доступен по адресу:
`https://ТВОЙ_USERNAME.github.io/ИМЯ_РЕПОЗИТОРИЯ/`

## Что делать дальше

**Обновление сайта:** после любых правок в коде:
```bash
git add .
git commit -m "Описание изменений"
git push
```

GitHub Pages обновит сайт автоматически (обычно в течение 1–2 минут).

---

## Потенциальные сложности

1. **Имя папки с пробелами** — в командах путь к папке «Руслан Бикбулатов» нужно брать в кавычки: `"Руслан Бикбулатов"`.
2. **Авторизация** — если `git push` просит логин/пароль, GitHub с 2021 года не принимает обычный пароль. Нужен Personal Access Token или вход через SSH.
3. **Русские символы** — Git и GitHub нормально работают с кириллицей в именах файлов и папок.
